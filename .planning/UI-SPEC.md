# Kinship — Unified UI Design Contract (Miro Redesign)

> Visual and interaction contract for the Miro-inspired redesign of all three implemented phases.
> Covers Phase 1 (Foundation & Onboarding), Phase 2 (Home Chores), Phase 3 (Kids Activities + Calendar).
> Generated by gsd-ui-researcher. Consumed by gsd-planner and gsd-executor.

---

## 1. Design System

| Property | Value |
|----------|-------|
| Tool | shadcn (style: base-nova, base color: neutral) |
| Preset | base-nova |
| Component library | @base-ui/react |
| Icon library | lucide-react |
| Font (display/headings) | Space Grotesk — CSS var `--font-display` |
| Font (body/labels) | Noto Sans — CSS var `--font-body` |
| CSS approach | Tailwind v4 CSS-first — tokens in `@theme inline {}` in `src/app/globals.css` |
| Inspiration | Miro design system (see `/DESIGN.md`) |

Space Grotesk is the Google Fonts substitute for Miro's proprietary Roobert PRO. Noto Sans is used
verbatim (same as Miro). Both are loaded via `next/font/google`.

## 2. Font Configuration

### layout.tsx replacement

Replace current `Plus_Jakarta_Sans` + `Public_Sans` imports with:

```tsx
import { Space_Grotesk, Noto_Sans } from "next/font/google";

const spaceGrotesk = Space_Grotesk({
  variable: "--font-display",
  subsets: ["latin"],
  weight: ["400", "600"],
});

const notoSans = Noto_Sans({
  variable: "--font-body",
  subsets: ["latin"],
  weight: ["400", "600"],
  style: ["normal"],
});
```

Apply to `<html>` className:

```tsx
className={`${spaceGrotesk.variable} ${notoSans.variable} h-full antialiased`}
```

**OpenType features** (apply via `font-feature-settings` in globals.css):

```css
/* Display headings — Space Grotesk with Miro-equivalent variants */
.font-display, [class*="font-display"] {
  font-feature-settings: "cv03", "cv04", "cv09", "cv11";
  letter-spacing: -0.02em; /* Miro's -1.68px at 56px scales to -0.02em */
}

/* Body — Noto Sans with Miro stylistic sets */
.font-body, [class*="font-body"] {
  font-feature-settings: "liga" 0, "ss01", "ss04", "ss05";
}
```

## 4. Spacing Scale

Declared values — all multiples of 4:

| Token | Value | Tailwind class | Usage |
|-------|-------|----------------|-------|
| xs | 4px | `p-1` / `gap-1` | Icon gaps, badge internal padding, tight label pairs |
| sm | 8px | `p-2` / `gap-2` | Spacing between icon and label text; inline badge padding |
| md | 16px | `p-4` / `gap-4` | Default form field gaps; card internal padding; input padding |
| lg | 24px | `p-6` / `gap-6` | Card padding for content cards; section padding |
| xl | 32px | `p-8` / `gap-8` | Layout gaps between major content blocks |
| 2xl | 48px | `p-12` / `gap-12` | Section breaks — between page header and content |
| 3xl | 64px | `p-16` / `gap-16` | Page-level vertical padding; hero sections |

**Exceptions:**
- Touch targets (checkboxes, row action buttons, mobile nav items): minimum 44px height. Use `min-h-[44px]` with centered content.
- Onboarding wizard card container: 40–50px border-radius (`rounded-[40px]`), internal padding 48px (`p-12`).
- Auth page card: 24px radius (`rounded-2xl`), 32px internal padding (`p-8`).

## 5. Typography

Space Grotesk replaces Roobert PRO for all display/heading roles.
Noto Sans is the body font verbatim from Miro.

| Role | Font | Class | Size | Weight | Line Height | Letter Spacing | Usage |
|------|------|-------|------|--------|-------------|----------------|-------|
| Heading | Space Grotesk | `font-display` | 32px | 600 | 1.2 | -0.02em | Page titles, modal titles, onboarding titles, auth hero tagline, card headers |
| Body Standard | Noto Sans | `font-body` | 16px | 400 | 1.5 | -0.01em | Descriptions, form fields, sub-headings (weight 600), section labels |
| Label | Noto Sans | `font-body` | 14px | 400 | 1.4 | normal | Badge text, form field labels, metadata (date, assignee, area), nav items |
| Caption | Noto Sans | `font-body` | 12px | 400 | 1.15 | normal | Micro labels, time-ago, step counters, nav section labels |

**Size mapping:** Use Heading (32px) for primary focal text on any view. Use Body (16px) at weight 600 for sub-headings, secondary titles, and calendar month/year labels. Use Label (14px) for all metadata. Use Caption (12px) for all micro/secondary labels.

**Weights:** Only 2 weights in use — **400** (body, labels, captions) and **600** (headings, sub-headings, button text, emphasis). Do not use weight 500, 700, or 800.

**Rules:**
- Never use pure `#000000`. Always use `--color-kinship-on-surface` (`#1c1c1e`) for body text.
- Secondary text (metadata, muted labels) uses `--color-kinship-on-surface-variant` (`#555a6a`).
- Placeholder text uses `--color-kinship-placeholder` (`#a5a8b5`).
- Maximum 2 type sizes on any single card or row.
- Heading always `font-display` (Space Grotesk); body/labels always `font-body` (Noto Sans).

## 6. Color Contract

### 60/30/10 split

| Role | Token | Hex | Percentage | Usage |
|------|-------|-----|-----------|-------|
| Dominant | `--color-kinship-canvas` | `#ffffff` | 60% | Card backgrounds, modal surfaces, white canvas areas |
| Near-dominant | `--color-kinship-surface` | `#f5f5f7` | (within 60%) | Page-level background, replacing the old `#f7f9fb` |
| Secondary | `--color-kinship-surface-container` | `#f0f0f2` | 30% | Filter bars, sidebar bg, section separators, input backgrounds |
| Accent | `--color-kinship-primary` | `#5b76fe` | 10% | Specific interactive elements only — see reserved list |
| Destructive | `--color-kinship-destructive` | `#e53e3e` | — | Destructive action buttons and confirmation dialogs only |

### Accent (`#5b76fe`) reserved for

1. Primary CTA button fills (e.g. "Add Task", "Add Activity", "Create household")
2. Focus ring on inputs and interactive controls
3. Unread notification badge background
4. Active/selected nav item indicator
5. Calendar event dot color for the Chores module
6. Progress indicator in onboarding wizard steps
7. "Mark complete" checkbox active state

Accent is NOT used on: section borders, general hover backgrounds, informational text, inactive badges.

### Module accent palette

| Module | Light surface | Dark text | Dot/indicator | Use |
|--------|--------------|-----------|---------------|-----|
| Chores | `#c3faf5` (teal) | `#187574` | `#187574` | Dashboard card header tint, status badge bg, calendar dot |
| Kids Activities | `#ffc6c6` (coral) | `#600000` | `#e05252` | Dashboard card header tint, category badge bg, calendar dot |
| Calendar (standalone) | `#ffd8f4` (rose) | `#7a1060` | `#b34a9c` | Calendar page accent, calendar module icon tint |

**Rule:** Never mix more than 2 module pastel accents in a single view section.

### No-Line Rule (preserved from original spec)

1px solid borders are prohibited for sectioning content. Background color shifts define boundaries:
- `#f5f5f7` page canvas → `#ffffff` card = natural separation via elevation contrast
- Filter bars: `#f0f0f2` container background vs `#ffffff` card background
- Row separation within lists: use 8px vertical gap (`gap-2`) — no divider lines

**Exception:** 1px `border-border` (`#e0e2e8`) is permitted on:
- Input and select fields (bottom border or full outline)
- Dialog/modal border (`ring-miro` shadow replaces solid border)
- Outlined secondary buttons

### Elevation

Miro minimal elevation — no heavy `box-shadow`. Two allowed elevation levels:

| Level | CSS | Usage |
|-------|-----|-------|
| Ring (level 1) | `box-shadow: rgb(224,226,232) 0px 0px 0px 1px` | Cards, modals, popovers — standard |
| Float (level 2) | `box-shadow: 0 20px 40px rgba(28,28,30,0.06)` | Dragging states, dropdown menus |

| Icon-only | transparent | none | `#555a6a` | `rounded-lg` | `p-2` | Filter toggles, kebab menus, close buttons — **must always include `aria-label`** (e.g. `aria-label="More options"`, `aria-label="Close dialog"`, `aria-label="Previous month"`) |

**Hover state:** Primary button → `#2a41b6` (actionable-pressed). Outline → background `#f0f0f2`.

**Minimum touch target:** All button variants — `min-h-[44px]` on mobile.

## 10. Accessibility Contract

- All interactive elements keyboard-navigable; Tab order follows visual reading order.
- Color is never the sole indicator: all badges, dots, and status indicators include a text label or aria-label.
- Module accent colors (teal/coral/rose) meet WCAG AA contrast when dark variant text is used on light pastel backgrounds.
- Blue 450 (`#5b76fe`) on white: contrast ratio 3.1:1 — acceptable for large text and UI components (WCAG AA non-text). Do NOT use Blue 450 for body text below 18px.
- Focus ring: `#5b76fe 0 0 0 3px` on all interactive elements — always visible regardless of hover state.
- Minimum 44px touch target on all interactive row elements.
- Dialog focus trap: Escape closes; focus returns to trigger element on close.
- Child tab row: `role="tablist"`, each tab `role="tab"`, panel `role="tabpanel"`.
- Calendar grid: each day cell `role="gridcell"`, `aria-label="[Day name], [Month] [Date]"`.
- Notification bell: `aria-label="Notifications, {N} unread"` (or "Notifications" when zero).
- Mark complete checkbox: `aria-label="Mark [Task name] complete"`, `aria-checked` state.
