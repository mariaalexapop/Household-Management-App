<!-- Atomic invite claim pattern (KEY CONSTRAINT from prompt) -->
```sql
-- Must use single UPDATE with RETURNING to prevent race condition
UPDATE household_invites
SET claimed_at = NOW(), claimed_by = $userId
WHERE token = $token
  AND claimed_at IS NULL
  AND expires_at > NOW()
RETURNING *
```

  <behavior>
    - Email invite POST route: verifies caller is admin of householdId (SELECT role FROM household_members WHERE user_id=caller AND household_id=target AND role='admin'), calls inviteUserByEmail, enqueues send-invite-email Inngest job
    - Link invite POST route: generates UUID token, inserts household_invites row with 7-day expiry, returns { inviteUrl: `${APP_URL}/join/${token}` }
    - Accept invite GET route: performs atomic UPDATE WHERE claimed_at IS NULL RETURNING *, if 0 rows affected returns 409 "Link already used or expired", if 1 row: inserts household_members (role='member'), redirects to /dashboard
    - Member DELETE route: verifies caller is admin, deletes household_members row (cannot delete self if only admin)
    - send-invite-email function: receives { email, householdName, inviteUrl }, sends email via Resend
  </behavior>
  <action>
    Create src/lib/inngest/client.ts:
    - Export inngest = new Inngest({ id: 'household-app' })

    Create src/lib/inngest/functions/send-invite-email.ts:
    - inngest.createFunction({ id: 'send-invite-email' }, { event: 'household/invite.created' }, ...)
    - Handler receives event.data: { email: string, householdName: string, inviteUrl?: string }
    - Creates new Resend(process.env.RESEND_API_KEY!) instance
    - Sends email: from RESEND_FROM_EMAIL, to email, subject "You've been invited to join {householdName}", HTML body with invite link (if present) or message to check email for Supabase magic link
    - If RESEND_API_KEY not set: log warning and skip (do not throw — dev environments may not have email configured)

    Create src/app/api/inngest/route.ts:
    - Export GET, POST, PUT from serve({ client: inngest, functions: [sendInviteEmail] })

    Create src/app/api/household/invite/route.ts (POST):
    - Authenticate: use createClient from server.ts, call getUser()
    - Parse body: { email: string, householdId: string }
    - Verify admin: query household_members WHERE user_id=caller AND household_id=householdId AND role='admin'
    - If not admin: return 403
    - Call createAdminClient().auth.admin.inviteUserByEmail(email, { data: { household_id: householdId }, redirectTo: APP_URL + '/auth/accept-invite' })
    - Enqueue Inngest event: inngest.send({ name: 'household/invite.created', data: { email, householdName } })
    - Return 200 { success: true }

    Create src/app/api/household/invite/link/route.ts (POST):
    - Authenticate + verify admin (same pattern)
    - Parse body: { householdId: string }
    - Insert into household_invites: { householdId, token: crypto.randomUUID(), invitedBy: userId, expiresAt: 7 days from now, email: null }
    - Return 200 { inviteUrl: `${APP_URL}/join/${token}` }

    Create src/app/api/household/invite/accept/route.ts (GET):
    - Extract token from searchParams
    - Authenticate current user (they must have completed signup first, then hit this URL)
    - Execute the atomic claim using Drizzle's sql template or db.execute:
      UPDATE household_invites SET claimed_at = NOW(), claimed_by = userId WHERE token = tokenParam AND claimed_at IS NULL AND expires_at > NOW() RETURNING *
    - If result.rowCount === 0: return redirect('/auth/login?error=invite_invalid')
    - Insert into household_members: { householdId: result.rows[0].household_id, userId, role: 'member', displayName: user.email }
    - Redirect to /dashboard

    Create src/app/api/household/members/[memberId]/route.ts (DELETE):
    - Authenticate + verify admin
    - Prevent admin from deleting themselves if they are the only admin
    - Delete from household_members WHERE id = memberId AND household_id matches caller's household
    - Insert into activity_feed: { householdId, actorId: callerId, eventType: 'member_removed', entityType: 'household_member', entityId: memberId }
    - Return 200 { success: true }

    Create tests/unit/household/invite.test.ts:
    - Test: invite link token is a valid UUID
    - Test: expired invite (expires_at in the past) returns 410 gone
    - Test: already-claimed invite (claimed_at IS NOT NULL) returns 409 conflict
    - These tests mock the database calls using vi.mock — do not require live DB connection
