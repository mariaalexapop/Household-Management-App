  <name>Create Supabase project in EU Frankfurt region</name>
  <action>
    1. Go to https://supabase.com and sign in (or create account)
    2. Click "New project"
    3. CRITICAL: Set Region to "Frankfurt (eu-central-1)" — this is required for GDPR (PLAT-04). It cannot be changed after creation.
    4. Set a strong database password and save it securely
    5. Wait for project to finish provisioning (~2 minutes)
    6. Go to Project Settings → API:
       - Copy "Project URL" → this is NEXT_PUBLIC_SUPABASE_URL
       - Copy "anon / public" key → this is NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY
       - Copy "service_role" key (click reveal) → this is SUPABASE_SERVICE_ROLE_KEY
    7. Go to Project Settings → Database → Connection String → URI mode (NOT "Session mode"):
       - Copy the direct connection URI → this is DATABASE_URL (use postgres:// scheme)
    8. Go to Authentication → Providers → Google → Enable it:
       - You'll need a Google Cloud Console project with OAuth 2.0 credentials
       - Authorized redirect URI to add: https://[your-project-ref].supabase.co/auth/v1/callback
       - Copy Client ID and Client Secret into Supabase
    9. Create .env.local at the project root with all four values (see env_vars above)
  </action>
  <done>
    .env.local exists with NEXT_PUBLIC_SUPABASE_URL, NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY, SUPABASE_SERVICE_ROLE_KEY, and DATABASE_URL all set.
    Supabase dashboard shows region as "Frankfurt (eu-central-1)".
    Google OAuth provider shows as enabled in Supabase Authentication → Providers.
  </done>

       When prompted: TypeScript=yes, style=default, base color=slate, globals.css=src/app/globals.css, CSS variables=yes, aliases as @/components @/lib etc.
       Install initial components: npx shadcn@latest add button card input label badge avatar separator

    4. Install tw-animate-css (NOT tailwindcss-animate):
       pnpm add tw-animate-css

    5. Update src/app/globals.css:
       - Keep the @import "tailwindcss" at the top (Tailwind v4 CSS-first)
       - Add @import "tw-animate-css" AFTER the tailwind import

       - DO NOT create tailwind.config.ts — Tailwind v4 is CSS-first; config file does not exist

    6. Create src/lib/supabase/client.ts:
       ```typescript
       import { createBrowserClient } from '@supabase/ssr'
       export function createClient() {
         return createBrowserClient(
           process.env.NEXT_PUBLIC_SUPABASE_URL!,
           process.env.NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY!
         )
       }
       ```

    7. Create src/lib/supabase/server.ts (async, uses cookies() from next/headers):
       ```typescript
       import { createServerClient } from '@supabase/ssr'
       import { cookies } from 'next/headers'
       export async function createClient() {
         const cookieStore = await cookies()
         return createServerClient(
           process.env.NEXT_PUBLIC_SUPABASE_URL!,
           process.env.NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY!,
           {
             cookies: {
               getAll() { return cookieStore.getAll() },
               setAll(cookiesToSet) {
                 try {
                   cookiesToSet.forEach(({ name, value, options }) =>
                     cookieStore.set(name, value, options)
                   )
                 } catch {}
               },
             },
           }
         )
       }
       ```

    8. Create src/lib/supabase/admin.ts (service role — NEVER expose to browser):
       ```typescript
       import { createClient } from '@supabase/supabase-js'
       export function createAdminClient() {
         return createClient(
           process.env.NEXT_PUBLIC_SUPABASE_URL!,
           process.env.SUPABASE_SERVICE_ROLE_KEY!,
           { auth: { autoRefreshToken: false, persistSession: false } }
         )
       }
       ```

    9. Create middleware.ts using the EXACT pattern from 01-RESEARCH.md Pattern 1:

       - Redirect unauthenticated users away from protected routes to /auth/login
       - Do NOT redirect from /auth/* routes
       - Export a config matcher that excludes _next/static, _next/image, favicon.ico, and api routes that don't need auth
