  <behavior>
    - Zustand store: setHouseholdType replaces previous type; toggleModule adds if absent, removes if present
    - createHousehold Server Action: validates inputs with Zod, runs db.insert for households → inserts household_members with role='admin' and current user_id → inserts household_settings with household_type and active_modules
    - Returns { success: false, error: "..." } if user already has a household (prevent double-create)
    - Returns { success: true, householdId: "..." } on success
    - Uses Drizzle transaction (db.transaction) so all three inserts are atomic
    - After successful insert, redirects to /dashboard (use redirect() from next/navigation inside the action or return success and redirect from the component)
  </behavior>
  <action>
    Create src/stores/onboarding.ts using Zustand create():
    - State: step (default 1), householdName (''), householdType (null), activeModules ([])
    - Actions: setStep, setHouseholdName, setHouseholdType, toggleModule, reset
    - Do NOT persist to localStorage — wizard state is ephemeral (lost on refresh is acceptable; user can start again)

    Create src/lib/validations/onboarding.ts with:
    - step1Schema: householdName (string min 2 max 50), householdType (z.enum(['couple', 'family_with_kids', 'flatmates', 'single']))
    - step2Schema: activeModules (z.array(z.enum(['chores', 'car', 'insurance', 'electronics', 'kids'])).min(1, 'Select at least one module'))
    - createHouseholdSchema: step1Schema.merge(step2Schema)

    Create src/app/actions/onboarding.ts:
    'use server'
    - Import createClient from src/lib/supabase/server.ts to get current user via supabase.auth.getUser()
    - Import db and schema tables from src/lib/db
    - Validate input with createHouseholdSchema.safeParse()
    - Check if user already has a household: SELECT from household_members WHERE user_id = userId LIMIT 1
    - If exists: return { success: false, error: 'Household already exists' }
    - Use db.transaction(async (tx) => { ... }) to:
      1. INSERT into households → get householdId
      2. INSERT into household_members with { householdId, userId, role: 'admin', displayName: user.email (initial name) }
      3. INSERT into household_settings with { householdId, householdType, activeModules }
    - On success: return { success: true, householdId }
    - Do NOT call redirect() inside the Server Action — return success and let the component handle redirect
  </action>

  </verify>
