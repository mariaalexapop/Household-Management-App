Revalidate '/insurance', '/calendar'.

IMPORTANT: The deletePolicy action must clean up both Storage files AND document DB rows before deleting the policy, to avoid orphaned files (Pitfall 3 from research).

- Trigger: `insurance/expiry.reminder.scheduled`
- Event data: `{ policyId, householdId, expiryDate, reminderDays, createdBy }`
- Sleep until `expiryDate - reminderDays * days`
- Re-check: query insurancePolicies to verify expiryDate unchanged and policy still exists
- Notification message: `"{insurer} {policyType} policy expires in {reminderDays} days"`
- Notification type: 'insurance_expiry_reminder'
- Email link to /insurance

**2. `sendInsurancePaymentReminder`**
- Trigger: `insurance/payment.reminder.scheduled`
- Event data: `{ policyId, householdId, nextPaymentDate, reminderDays, createdBy }`
- Sleep until `nextPaymentDate - reminderDays * days`
- Re-check: query insurancePolicies to verify nextPaymentDate unchanged
- Notification message: `"{insurer} premium payment due in {reminderDays} days"`
- Notification type: 'insurance_payment_reminder'
- Email link to /insurance

Both functions: re-check DB before sending (same pattern as car reminder). If date has changed or policy deleted, return `{ skipped: true }`. Register both in the Inngest serve handler.
