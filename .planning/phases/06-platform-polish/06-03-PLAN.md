  <action>
**1. Install @next/bundle-analyzer:**

```bash
npm install -D @next/bundle-analyzer
```

**2. Update `next.config.ts` to integrate bundle analyzer:**

```typescript
import type { NextConfig } from 'next'
import path from 'path'
import withBundleAnalyzer from '@next/bundle-analyzer'

const withAnalyzer = withBundleAnalyzer({
  enabled: process.env.ANALYZE === 'true',
})

const nextConfig: NextConfig = {
  serverExternalPackages: ['pdf-parse', 'pdfjs-dist'],
  experimental: {
    serverActions: {
      bodySizeLimit: '4mb',
    },
  },
  turbopack: {
    root: path.resolve(__dirname),
  },
}

export default withAnalyzer(nextConfig)
```

**3. Add analyze script to package.json:**

Add to the "scripts" section:
```json
"analyze": "ANALYZE=true next build"
```

**4. Run `next build` and analyze output:**

Run a production build: `npm run build`

Examine the build output for:
- Route sizes (any route over 200KB first-load JS is concerning)
- Shared chunk sizes
- Total first-load JS for key routes (/dashboard, /chores, /calendar)

**5. Run Lighthouse CLI audit:**

```bash
# Start production server
npm run build && npm start &
# Wait for server to be ready, then run Lighthouse
npx lighthouse http://localhost:3000/dashboard --output=json --output-path=./lighthouse-report.json --chrome-flags="--headless --no-sandbox" --only-categories=performance,pwa
```

Examine the Lighthouse report for:
- Performance score (target: 80+)
- PWA score (target: all installability checks pass)
- LCP (target: < 2.5s)
- CLS (target: < 0.1)
- FCP (target: < 1.8s)

**6. Fix identified issues.** Common fixes:

- **Large client bundles:** Check if any page imports heavy server-only libraries on the client side. Add `'use server'` directives or dynamic imports with `next/dynamic` for heavy components.
- **Unoptimized images:** Replace any `<img>` tags with Next.js `<Image>` component.
- **CLS from fonts:** Already handled by next/font (Space_Grotesk + Noto_Sans).
- **Large date-fns locale bundles:** If date-fns is importing all locales, configure tree-shaking.
- **Unnecessary client components:** If any large component tree is wrapped in `'use client'` but doesn't need interactivity, split into server + client parts.

**7. Run bundle analyzer for documentation:**

```bash
ANALYZE=true npm run build
```

This generates `.next/analyze/client.html` — document any chunks over 100KB in the SUMMARY.

**8. Clean up:** Remove lighthouse-report.json after extracting findings. Keep the analyze script in package.json for ongoing use.
