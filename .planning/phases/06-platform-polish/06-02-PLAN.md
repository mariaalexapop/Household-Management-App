Systematically audit every page at 375px viewport width. For each page, check:

1. No horizontal scroll (no element overflows viewport)
2. All text readable without zooming
3. All buttons/links have 44px+ touch targets (min-h-11 min-w-11 or equivalent padding)
4. Forms stack vertically on mobile (grid-cols-1 on small screens)
5. Modals/dialogs are near-full-screen on mobile
6. Tables scroll horizontally within their container (not the page)
7. Images don't overflow their container
8. Date pickers are usable on touch devices

**Common fixes to apply per page:**

**Dashboard (`dashboard/page.tsx`):**
- Ensure module cards grid is `grid-cols-1 sm:grid-cols-2 lg:grid-cols-3`
- Reduce padding on mobile: `p-4 sm:p-6 lg:p-8`
- Ensure quick-action buttons are touch-friendly (min-h-11)

**Chores (`chores/ChoresClient.tsx`):**
- Task form fields stack vertically: `grid grid-cols-1 sm:grid-cols-2 gap-4`
- Task cards should be full-width on mobile
- Filter bar wraps on mobile: `flex flex-wrap gap-2`

**Kids (`kids/KidsClient.tsx`):**
- Activity form stacks vertically on mobile
- Child tabs scroll horizontally if many children: `overflow-x-auto`
- Activity cards full-width on mobile

**Calendar (`calendar/CalendarClient.tsx`):**
- Month grid cells may be too small at 375px — ensure minimum tap target
- Week view should scroll horizontally if needed
- Event text should truncate with ellipsis, not overflow
- Filter chips wrap: `flex flex-wrap gap-2`

**Cars (`cars/CarsClient.tsx`):**
- Complex forms with date pickers — stack ALL fields vertically on mobile
- Service history timeline full-width
- Ensure react-day-picker calendar grid fits within 375px (it's ~280px wide, should be fine)
- Button groups stack or wrap on mobile: `flex flex-col sm:flex-row gap-2`

**Insurance (`insurance/InsuranceClient.tsx`):**
- Policy cards full-width on mobile
- Document upload area full-width
- Payment schedule table: `overflow-x-auto` on container

**Electronics (`electronics/ElectronicsClient.tsx`):**
- Item cards full-width on mobile
- Warranty info stacks vertically
- Document list full-width

**Costs (`costs/CostsClient.tsx`):**
- Data table must be in `overflow-x-auto` container (likely already is)
- Summary cards: `grid-cols-1 sm:grid-cols-2 lg:grid-cols-4`
- Chart containers should be responsive (width 100%)

**Household (`household/page.tsx`):**
- Member list full-width
- Invite form stacks vertically
- Action buttons touch-friendly

**Settings/Modules (`settings/modules/page.tsx`):**
- Module toggle cards full-width on mobile
- Sufficient padding and touch targets on toggles

**Chatbot (`ChatbotDock.tsx` and `ChatbotFab.tsx`):**
- ChatbotDock should be full-screen on mobile (inset-0 below sm breakpoint): `fixed inset-0 sm:bottom-4 sm:right-4 sm:w-[420px] sm:h-[600px] sm:inset-auto`
- ChatbotFab should have safe-area-bottom offset on notched phones: add `safe-area-bottom` class or `bottom: max(1rem, env(safe-area-inset-bottom))`
- Ensure FAB doesn't overlap bottom navigation or critical page content

**AppHeader (`nav/AppHeader.tsx`):**
- Ensure hamburger menu button is 44px+ touch target
- Title truncates on narrow screens: `truncate`
- Back button + title fit within 375px

**General patterns to apply everywhere:**
- Replace any `px-8` or `px-6` containers with `px-4 sm:px-6 lg:px-8`
- Replace `gap-6` in form layouts with `gap-4 sm:gap-6`
- Ensure all `<Button>` components have at least `h-11` (44px) on mobile
- Any side-by-side layout that doesn't fit 375px: add `flex-col sm:flex-row` or `grid-cols-1 sm:grid-cols-2`
