## Known Tech Debt

**RLS self-referential policy** — `household_members` SELECT policy uses a subquery on the same table, causing the Supabase browser client to always return empty. All queries go through Drizzle (server-side) as workaround. Fix in Phase 2: change policy to `user_id = auth.uid()` + security-definer function for cross-member reads.
