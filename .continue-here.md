## Key Technical Context

### Known Test Environment Quirks (critical to remember)
- **React 19 renders async in jsdom** — use `findByText()` not `getByText()` in all component tests
- **`lucide-react` v1.x breaks jsdom** — mock in tests: `vi.mock('lucide-react', () => ({...}))`
- **`@base-ui/react` (used in shadcn Badge) breaks jsdom** — mock: `vi.mock('@/components/ui/badge', () => ({...}))`
- `vitest.config.ts` has `resolve.dedupe: ['react', 'react-dom']` as baseline fix

### Architecture Decisions Locked In
- Drizzle `db` client (direct Postgres) bypasses RLS — auth via Supabase `getUser()` before all DB writes
- Zustand store is ephemeral (no localStorage) — onboarding wizard state lost on refresh by design
- Server Actions return `{ success, error }` — components handle `router.push()`, never `redirect()` inside actions
- Invite claim is atomic via raw SQL `UPDATE … WHERE claimed_at IS NULL RETURNING *` — no Drizzle ORM for this specific query

### File Conventions
- Route groups: `(app)` for authenticated pages, `(onboarding)` for wizard
- Server Components fetch data; Client Components handle interactivity
- Auth pattern: `const supabase = await createClient(); const { data: { user } } = await supabase.auth.getUser()`
